# File Splitting Rule

## 禁止
- 分割に便乗した無関係変更、設計変更、最適化、命名一括変更を行わない
- 分割のためだけに依存関係を追加しない

## 発動条件
- file が大きく見落としや衝突が起きやすい場合
- 関数が大きく複数責務が混在する場合
- 重複実装があり修正漏れリスクがある場合
- 分岐追加が増え条件漏れが起きやすい場合
- 行数と分岐数は目安とし、file 400-800 行、関数 80-150 行、主要分岐 5-10 件超を分割検討目安とする

## 分割単位
- 1 file は 1 概念に寄せる
- 公開 interface は最小にする
- 表示、入力、状態更新、外部副作用を混在させない

## 依存方向
- 入出力層 -> 中核ロジック層 -> 副作用層 の一方向とする
- 逆方向参照を禁止する
- 共有型は中核ロジック層に置く

## 手順
1. 新規 file を作成する
2. 既存の関数や構造を中身を変えずに移動する
3. 呼び出し側参照のみ修正する
4. 対象差分に応じた最小確認を行う
5. 安全な自動確認が無い場合は未実施理由と不足条件を明記する
6. 整形や引数整理が必要な場合は確認成功後の別作業に分離する

## 完了条件
- 最小確認または文面整合と参照整合の確認が成功している
- 分割理由を 1-2 文で記録している
- 記録には危険要因と回避内容を含める
